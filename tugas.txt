1.	Arsitektur Sistem
Aplikasi terdiri dari minimal tiga service terpisah:
•	Auth Service: mengelola login, token JWT, dan role (admin/user).
•	Main Service / Project Service: menangani data utama (sesuai tema proyek).
•	Gateway / Frontend Service: sebagai penghubung antar-service dan antarmuka pengguna.
Semua service harus berjalan dalam container terpisah dan saling berkomunikasi menggunakan Docker network.
Deployment akhir dilakukan ke Microsoft Azure (boleh menggunakan Azure App Service for Containers, Azure Container Apps, atau AKS).
2.	Database
Gunakan dua database terpisah:
•	User Database: untuk data akun, login, dan peran (role).
•	Project Database: untuk data utama proyek (misalnya siswa, pasien, buku, stok barang, dll.).
Jenis database bisa dipilih (SQL atau NoSQL).
Koneksi dan kredensial database harus disimpan dengan aman (tidak hard-coded di source code).
3.	Role dan Akses Pengguna
Sistem harus memiliki dua jenis akun:
•	Admin : dapat melakukan semua oeprasi CRUD pada semua data.
•	User : hanya dapat melakukan Read dan Update pada data milik sendiri.
Gunakan autentikasi berbasis JWT (JSON Web Token) dan sertakan validasi setiap endpoint sesuai role pengguna.
4.	Fitur Minimal
•	CRUD lengkap.
•	Login, register, dan logout.
•	Validasi input dan error handling (gunakan HTTP status code yang sesuai).
•	Tampilan web dengan UI/UX yang mudah digunakan.
•	Dokumentasi endpoint API di README.
•	Deployment di Microsoft Azure.
Aspek	Bobot	Kriteria Penilaian
1. Fungsionalitas & Arsitektur	25%	Aplikasi berfungsi lengkap (CRUD, login, role) dengan arsitektur microservices yang benar.
2. Deployment ke Azure	15%	Semua service berjalan di Azure, URL publik aktif, konfigurasi environment & secrets aman.
3. Security & Authorization	10%	Implementasi JWT, role-based access, validasi input, dan proteksi endpoint sudah tepat.
4. Error Handling & Logging	10%	Penanganan error konsisten, kode HTTP sesuai, pesan error informatif, logging tersedia.
5. UI/UX (Antarmuka Pengguna)	10%	Tampilan menarik, navigasi jelas, validasi input, pesan responsif.
6. Kualitas Kode & Struktur Proyek	10%	Struktur rapi, modular, komentar jelas, repo tertata, dokumentasi build lengkap.
7. Dokumentasi & Laporan	10%	README, diagram arsitektur, penjelasan endpoint, dan langkah deploy lengkap.
8. Presentasi & Demo	10%	Demo berjalan lancar, penjelasan teknis jelas, mampu menjawab pertanyaan dengan baik.

6. Sistem Data Pasien
Permasalahan:
Data pasien sering dicatat manual dan menyulitkan dokter untuk melacak riwayat.
Solusi aplikasi:
Sistem manajemen pasien dengan data dasar & dokter yang menangani.
Role:
Admin (petugas klinik): CRUD pasien & dokter.
User (pasien): melihat & memperbarui data dirinya.
